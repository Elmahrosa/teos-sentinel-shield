// deploy to vercel

Go live in
under 5 minutes.

You already have the repo at github.com/Elmahrosa/teos-sentinel-shield and the domain teos-sentinel-shield.vercel.app — here's the exact process to redeploy with this new landing page.

1
Add this file to your repo root
Save this index.html file into the root of your teos-sentinel-shield GitHub repo. Also add teos_sentinel.png (the logo) to the same root folder.
2
Add a vercel.json config (static site)
Create a file called vercel.json in the repo root with this content:
{ "rewrites": [{ "source": "/(.*)", "destination": "/index.html" }] }
3
Push to GitHub
Commit and push both files to your main branch. Vercel automatically detects the push and rebuilds.
git add index.html teos_sentinel.png vercel.json
git commit -m "feat: new landing page with logo and roadmap"
git push origin main
4
Vercel auto-deploys — check your dashboard
Go to vercel.com/dashboard → find project teos-sentinel-shield → click "Visit" to confirm it's live. Your URL stays the same: teos-sentinel-shield.vercel.app
5
If the project is not connected yet — import it
Go to vercel.com/new → "Import Git Repository" → select Elmahrosa/teos-sentinel-shield → Framework: Other (no build command needed for plain HTML) → Root directory: / → Deploy. Vercel will give you the same domain or you can set a custom one.
6
Optional: Set environment variables
If your bot backend needs env vars (bot token, Neon DB URL, Solana RPC), go to: Vercel Dashboard → Project → Settings → Environment Variables. Add each one. Then redeploy from the dashboard.
